# Remember - 현재 디렉토리에 기억 저장

사용자가 기억을 요청한 내용을 현재 작업 디렉토리에 `*-MEMORY.md` 파일로 저장하는 스킬.

## 두 가지 모드

### 모드 1: 특정 내용 기억 (대화 중 트리거)

사용자가 "기억해줘", "이거 기억해" 등의 표현과 함께 특정 내용을 언급하면, 해당 내용을 파일로 저장한다.

### 모드 2: 세션 요약 저장 (`/remember` 직접 호출)

사용자가 `/remember`를 직접 실행하면, 현재 세션의 대화 전체를 분석하여 요약본을 저장한다.

**세션 요약 절차:**

1. 현재 세션에서 수행한 작업들을 시간순으로 정리한다.
2. 다음 항목들을 포함하여 요약한다:
   - **작업 목적**: 이 세션에서 무엇을 하려고 했는지
   - **수행한 작업**: 실제로 한 일들 (생성/수정/삭제한 파일, 실행한 명령 등)
   - **주요 결정 사항**: 왜 특정 방식을 선택했는지
   - **미완료 사항**: 아직 끝나지 않은 작업이 있다면
   - **다음 단계**: 후속 세션에서 이어서 해야 할 것
3. 파일명은 `session-{YYYY-MM-DD}-{주제}-MEMORY.md` 형식으로 저장한다.
   - 예: `session-2026-03-19-remember-plugin-MEMORY.md`
4. 같은 날 같은 주제의 세션 파일이 있으면 내용을 추가한다.

**세션 요약 파일 형식:**

```markdown
# 세션 요약: {주제}

> 날짜: {YYYY-MM-DD}

## 작업 목적

{이 세션의 목표}

## 수행한 작업

- {작업 1}
- {작업 2}
- ...

## 주요 결정 사항

- {결정 1}: {이유}
- ...

## 미완료 사항

- {남은 작업이 있다면}

## 다음 단계

- {후속 작업이 있다면}
```

## 동작 방식 (모드 1: 특정 내용 기억)

1. 사용자의 메시지에서 기억할 핵심 내용을 파악한다.
2. 내용에 맞는 적절한 영문 파일명을 결정한다. (예: `api-keys-MEMORY.md`, `deploy-process-MEMORY.md`)
3. 현재 작업 디렉토리(`CLAUDE_PROJECT_DIR` 또는 CWD)에 `{이름}-MEMORY.md` 파일로 저장한다.
4. 이미 같은 이름의 파일이 있으면 내용을 추가(append)하거나 업데이트한다.

## 파일명 규칙

- 모드 1: `{영문-kebab-case-이름}-MEMORY.md`
- 모드 2: `session-{YYYY-MM-DD}-{주제}-MEMORY.md`
- 내용을 잘 설명하는 간결한 이름을 선택한다.
- 예시 (모드 1):
  - DB 접속 정보 → `db-connection-MEMORY.md`
  - 배포 절차 → `deploy-process-MEMORY.md`
  - API 키 관리 → `api-keys-MEMORY.md`
  - 프로젝트 결정 사항 → `project-decisions-MEMORY.md`
  - 특정 버그 관련 → `auth-bug-MEMORY.md`

## 파일 형식 (모드 1)

```markdown
# {제목}

> 저장일: {YYYY-MM-DD}

{기억할 내용}
```

기존 파일에 추가할 때는 `---` 구분선과 함께 새 항목을 아래에 덧붙인다:

```markdown
# {제목}

> 저장일: {YYYY-MM-DD}

{기존 내용}

---

> 추가일: {YYYY-MM-DD}

{새로운 내용}
```

## 실행 절차

1. 모드를 판별한다: `/remember` 직접 호출이면 모드 2, 그 외에는 모드 1.
2. **모드 1**: 사용자의 요청에서 기억할 내용을 추출하고 주제에 맞는 파일명을 결정한다.
3. **모드 2**: 현재 세션의 대화를 분석하여 요약을 작성하고 `session-{날짜}-{주제}` 파일명을 결정한다.
4. 같은 이름의 `-MEMORY.md` 파일이 현재 디렉토리에 이미 있는지 Glob으로 확인한다.
5. 파일이 없으면 새로 생성하고, 있으면 기존 파일을 읽어서 내용을 추가한다.
6. 저장 완료 후 파일명과 저장된 내용을 사용자에게 간단히 알려준다.

## 주의사항

- 비밀번호, 토큰, 시크릿 같은 민감한 정보는 저장하지 않는다. 대신 사용자에게 경고하고 환경변수나 시크릿 매니저 사용을 권장한다.
- 파일은 반드시 현재 작업 디렉토리에 저장한다. 다른 경로에 저장하지 않는다.
- 내용은 미래의 Claude 세션이 읽어도 이해할 수 있도록 충분한 맥락을 포함한다.
- 세션 요약(모드 2)은 핵심만 간결하게 작성한다. 대화 내용을 그대로 복사하지 않는다.
